数据类型转换

自动类型转换

char–>int–>long–>float–>double

byte–>short–>int–>long–>float–>double

  • 多种类型混合运算，结果是容量最大的类型
  • （byte,short）不能与char自动转换
  • byte,short,char进行运算时，先转换成int，再进行运算
  • boolean不参与转换

强制类型转换

  • 精度大转换为精度小，需要强制类型转换，用强制类型转换符()
  • 强制类型转换符往往只对最近的操作数有效，往往需要用小括号来提升优先级
  • char类型可以保存int的常量值，但不可以保存int的变量值

基本数据类型和String类型的转换

  • 基本数据类型–>String：在基本数据类型后加””即可
  • String–>基本数据类型：
    Integer.parseInt(String变量)，就将String转换成int
    Double.parseDouble(String变量)，将String转为Double
  • String–>char，通常将字符串的第一个字符赋值给char
    str.charAt(0)，表示取出字符串变量str的第一个字符

类与对象

真正的对象其实是new Cat()，也就是创建出来的数据内存空间，而cat1是对象的名字。

类与对象的内存分配：

  • 栈：一般存放基本数据类型(局部变量)
  • 堆：存放对象
  • 方法区：常量池（常量，比如字符串），类加载信息（属性和方法）

构造器、构造方法

构造函数可以重载。

用来对象的初始化。

如果自己写了一个构造方法，则此方法覆盖了默认无参构造方法，就不能使用默认构造方法。如果想调用默认，则必须重新定义一下默认构造方法。

如果自己没有写构造方法，则调用默认构造方法。

对象创建流程（空间）

  • 加载类信息，只加载一次
  • 在堆中分配空间（地址）
  • 完成对象初始化：默认初始化–>显式初始化–>构造器初始化
  • 把对象在对中的地址返回给对象名

this

this表示当前对象

简单说，哪个对象调用了，this就指的是哪个对象

this.hashCode()可以简单的看成可以获取对象的地址

  • this用来区分属性和局部变量
  • this可以访问本类中的属性、方法、构造器
  • this只能在类定义的方法中使用
  • this通过this(参数列表)访问构造器，即只能在一个构造器内访问另一个构造器，且必须放在构造函数第一条。

访问修饰符

public：对外公开

protected：对子类和同一个包的类公开

默认：对同一个内的类公开

private：只有本类才可以访问

  • 访问修饰符可以修饰类、成员方法、属性
  • 只有默认的和public才可以修饰类

封装

封装的本质是用getter()和setter()方法，访问私有属性

将构造器与setter()结合，防止通过构造函数绕过封装

继承

  • 子类继承了所有的属性和方法，但是私有属性和方法不能在子类直接访问，要通过父类提供的公共方法去访问。
  • 子类必须必须调用父类的构造器，完成父类的初始化。
  • 当创建子类对象时，不管使用子类的哪个构造器，默认情况下总会去调用父类的无参构造器。如果父类没有无参构造器，则必须在子类的构造器中用super去指定使用父类的哪个构造器完成对父类的初始化工作，否则，编译不会通过。

super

  • 访问父类的属性，但不能访问父类的private属性
  • 访问父类的方法，但不能访问父类的private方法
  • 访问父类的构造器：super(参数列表)
    只能放在构造器的第一句，只能出现一句
  • 调用父类的构造器的好处：分工明确，父类属性由父类初始化，子类属性由子类初始化
  • 当子类中有和父类中的成员（属性和方法）重名时，为了访问父类的成员，必须通过super。

super和this的比较

重写

  • 子类方法的返回类型必须和父类一致，或者是父类返回类型的子类

如，父类返回类型是Object，子类返回类型是String。

  • 子类方法不能缩小父类方法的访问权限。

多态

多态提高代码复用性。

多态的前提是：两个对象（类）存在继承关系。

多态的本质是：父类的引用指向子类的对象（相当于把子类升级成父类）,也就是向上转型。

  • 编译类型看左边，运行类型看右边
  • 可以调用父类中的所有成员（需遵守访问权限），不能调用子类的特有成员。因为在编译阶段，是由编译器来决定哪些成员可以调用的。
  • 最终运行效果看子类的具体实现

在调用方法时，先从子类中找，如果没有，就找父类…

多态的向下转型

只能强转父类的引用，不能强转父类的对象。

要求父类的引用必须指向的是当前目标类型的对象。也即是先向上转型，再向下转型。

当向下转型后，就可以调用子类类型中的所有成员。

父类子类属性相同

向上转型时，获取的属性值取决于编译类型

instanceOf

判断obj对象的运行类型是否是B类型，或者是B类型的子类型

动态绑定机制（非常非常重要）

  • 当调用对象方法的时候，该方法会和对象的内存地址、运行类型绑定
  • 当调用对象的属性的时候，没有动态绑定机制，哪里声明，哪里使用。

输出：30，20

分析：

a.sum()

a的编译类型是A，运行类型是B。
因此先在B中找sum()，没有，再去父类A中找到sum()；
A中的sum()又调用了对象方法getI()，执行动态绑定机制，去运行类型B中找到getI()，得到i=20;
因此第一个输出为20+10=30。

a.sum1()

a的编译类型是A，运行类型是B。
因此先在B中找sum1()，没有，再去父类A中找到sum1()；
A中的sum()又调用了对象属性i，没有动态绑定机制，在A中声明了i，因此i=10;
因此第二个输出为10+10=20

多态数组

数组的定义类型为父类类型，里面保存的实际元素为子类类型。

如果多态数组调用子类的特有方法，那么可以把其类型强转为子类，再调用。

hashCode()

会针对不同的对象返回不同的整数。这一般是通过将对象的内部地址转换成一个整数实现的。

  • 可以提高具有哈希结构的容器的效率
  • 两个引用，如果指向的是同一个对象，则哈希值相同
  • 两个引用，如果指向的是不同对象，则哈希值不同

toString()

默认返回：全类名（包名+类名）+@+哈希值的十六进制

重写toString()，使用快捷键alt+insert -> toString

重写toString()一般是用来输出对象的属性，当然也可以自己修改

当直接输出一个对象时，toString()会被默认调用：

其本质就是调用toString()

finalize()

  • 对象被回收时，系统自动调用改对象的finalize()。子类就可以重写该方法，做一些释放资源的操作
  • 什么时候被回收：当某个对象没有任何引用时，则jvm就认为这个对象是一个垃圾对象，就会使用垃圾回收机制来销毁该对象，再销毁对象之前，会先调用finalize()
  • 垃圾回收机制的调用，是由系统来决定的，也可以通过System.gc()主动出发垃圾回收机制
